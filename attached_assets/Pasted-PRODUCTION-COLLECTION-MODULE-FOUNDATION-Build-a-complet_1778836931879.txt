PRODUCTION & COLLECTION MODULE FOUNDATION

Build a complete Production & Collection Management module for the rubber plantation ERP.

This module must integrate with the already existing:

- authentication system
- role system
- project-wise access
- dashboard architecture
- user profile system
- governance system
- lifecycle system

Do NOT rebuild existing systems.

Use existing authentication and project access architecture.

====================================================
CORE WORKFLOW
====================================================

This module has 2 completely separate workflows:

1. Collection Entry
2. Store Entry

Do NOT combine them.

Collection Entry:
- represents fresh sheets collected before drying

Store Entry:
- represents sheets moved into store after drying outside for 3–5 days

====================================================
IMPORTANT BUSINESS LOGIC
====================================================

Sheets do NOT have individual IDs.

This is a continuous rolling production process.

Daily:
- new sheets are collected
- old sheets continue drying outside
- some sheets later move into store

Therefore:

DO NOT implement:
- per-sheet tracking
- batch tracking for sheets
- aging flags for exact sheet age

Instead maintain only:

Pending outside store total:

Pending sheets =
Total collected sheets − Total stored sheets

- employee

- time

====================================================
PENDING OUTSIDE STORE ENGINE
====================================================

Build automatic calculations.

Pending outside store:

Total collected sheets − Total stored sheets

Example:

Collected total:
5100 sheets

Stored total:
4700 sheets

Pending:
400 sheets

Do NOT attempt exact sheet aging logic.

Do NOT create fake age flags.
