FIX PROJECT CARD DATA FETCHING AND MAKE PROJECT CARDS LIVE GOVERNANCE DATA NODES.

Current issue:
Project cards are not correctly fetching and displaying live project-linked data.

Example:
Stock data is missing, stale, incorrect, or disconnected from actual project stock records.

This indicates the project card is functioning as a static UI summary instead of a live governance-linked data node.

==================================================
CORE ARCHITECTURE RULE
==================================================

Project cards must NEVER use:
- hardcoded values,
- stale cache,
- temporary frontend state,
- disconnected summaries,
- manually entered totals.

Every displayed value must come directly from:
- live project-linked data sources,
- validated aggregation queries,
- real-time operational records.

==================================================
PROJECT CARD DATA ENGINE
==================================================

Implement a centralized:

PROJECT_CARD_DATA_AGGREGATION_ENGINE

This engine must dynamically fetch and aggregate data from:
- collection system
- store system
- stock ledger
- stock transfer system
- sales ledger
- reimbursement engine
- ownership engine
- LCA engine
- participant system
- agreement system
- audit system

==================================================
STOCK FETCHING RULES
==================================================

Project stock displayed on project cards must represent:

REAL CONSOLIDATED PROJECT STOCK

including:
- project stores
- central stores
- overflow stores
- transferred stock
- pending dispatch stock
- reserved stock
- buyer pickup pending stock

==================================================
IMPORTANT STOCK CALCULATION RULE
==================================================

Project stock must calculate using:

TOTAL INWARD
MINUS
TOTAL OUTWARD
MINUS
DISPATCHED STOCK
PLUS/MINUS
TRANSFER ADJUSTMENTS

grouped by:
- project ownership
NOT merely physical location.

==================================================
VERY IMPORTANT RULE
==================================================

Physical stock location must NEVER break project ownership accounting.

Even if stock moved to:
- central store
- overflow store
- another warehouse

project card must still show:
- correct project-owned stock.

==================================================

==================================================

Project cards must fetch:
- latest committed stock ledger state,
NOT:
- cached page memory,
- stale API snapshots,
- delayed frontend totals.
