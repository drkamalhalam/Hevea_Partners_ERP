LOGIN ACCESS & AUTHENTICATION REFINEMENT

IMPORTANT INSTRUCTIONS

This is an existing Plantation ERP.

The architecture audit identified an existing authentication system:

- Clerk Authentication
- users table
- role-based permissions
- requireAuth middleware
- canAccessProject authorization
- operational access controls

These assets are reusable.

DO NOT rebuild authentication from scratch.

Analyze existing implementation first.

Reuse compatible components wherever possible.

Architecture requirements take precedence over current implementation.

--------------------------------------------------
OBJECTIVE
--------------------------------------------------

Refine Login Access architecture to align with Person Registry.

Implement:

1. Login Access Management
2. Person-to-Login Linking
3. Authentication Hardening
4. Authorization Refinement
5. Account Lifecycle Controls
6. Auditability
